# 调试指南

MOYAG项目常见问题和调试方法。

## Claude Code 问题

### CLI Hang (Mige API)
**症状**: `claude` 命令无响应，卡在启动阶段
**原因**: v2.1.170 在 `ANTHROPIC_BASE_URL` 指向非Anthropic端点时hang
**解决**: 绕过CLI，直接用Python urllib调API（参考 Mige API 笔记）

### MCP连接失败
**症状**: `claude mcp list` 显示 gbrain 断开
**排查**: 
1. 检查Python路径: `ls ~/AppData/Local/hermes/hermes-agent/venv/Scripts/python.exe`
2. 手动测试: `echo '{"jsonrpc":"2.0","id":1,"method":"tools/list"}' | python gbrain_mcp.py`
3. 重新注册: `claude mcp remove gbrain && claude mcp add -s user gbrain -- python "..." "..."`
4. 查看日志: `~/.claude/debug/`

## Hermes 问题

### 会话膨胀
**症状**: 响应变慢、上下文溢出
**预防**: 消息数>200条时主动 `/new`
**参考**: [[10-知识积累]] 经验沉淀

### Cron不执行
**检查**: `hermes cron list` 查看状态
**排查**: 检查 `~/.hermes/logs/` 错误日志

## GBrain 问题

### 索引失败
**症状**: `gbrain_index` 返回 indexed: 0
**检查**: 
1. Vault路径: `~/Documents/GBrain Vault/`
2. Chroma数据库: `~/.gbrain_chroma/`
3. 手动索引: `python gbrain.py index`

### 断链
**检测**: `gbrain_broken_links`
**修复**: 创建缺失笔记或修正链接
**自动**: Broken Links Watchdog (Cron, 周一/四 08:00)

## 环境问题

### Windows路径
- Hermes工具用正斜杠: `C:/Users/EDY/...`
- Git Bash: `/c/Users/EDY/...`
- Python: 两者都接受

### Python版本
- Hermes venv: `C:/Users/EDY/AppData/Local/hermes/hermes-agent/venv/`
