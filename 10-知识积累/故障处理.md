# 故障处理

应急响应和故障恢复流程。

## P0: 服务不可用

### Hermes Gateway 宕机
```
1. 检查: hermes gateway status
2. 重启: hermes gateway restart
3. 验证: curl localhost:8000/health
4. 日志: tail -50 ~/.hermes/logs/gateway.log
```
**自动监控**: Gateway Health Monitor (每30分钟)

### ECS 不可达
```
1. Ping: ping 47.237.203.217
2. SSH: ssh root@47.237.203.217
3. 检查服务: systemctl status visual-agent
4. 重启: systemctl restart visual-agent
```

## P1: 数据丢失风险

### GBrain 索引损坏
```
1. 备份: cp -r ~/.gbrain_chroma ~/.gbrain_chroma.bak
2. 重建: cd ~/Documents/GBrain\ Vault && python gbrain.py index
3. 验证: python -c "from gbrain import GBrain; print(GBrain('.').stats())"
```

### Obsidian 笔记丢失
```
1. 检查Git: cd ~/Documents/GBrain\ Vault && git status
2. 恢复: git checkout -- <file>
3. 日志: git log --oneline -5
```
**自动备份**: Git Auto-Backup (每天23:00)

## P2: 性能下降

### 磁盘空间不足
```
1. 检查: python ~/AppData/Local/hermes/scripts/disk-watchdog.py
2. 清理Cron日志: hermes sessions prune --older-than 7
3. 清理Chroma: 压缩或重建索引
```
**自动监控**: Disk Space Monitor (每2小时, >85%报警)

### 会话数据库膨胀
```
1. 统计: hermes sessions stats
2. 清理: hermes sessions prune --older-than 30
3. 验证: hermes sessions stats
```
**自动清理**: Session Cleanup (每天03:00)

## 监控总览

| 监控项 | Cron任务 | 频率 | 通知 |
|--------|---------|------|------|
| Gateway | Gateway Health Monitor | 30min | 仅异常 |
| 磁盘 | Disk Space Monitor | 2h | >85%报警 |

| 孤立笔记 | GBrain Orphan Check | 周一 | 始终报告 |

| 索引 | GBrain Auto-Index | 30min | 本地 |
